Design Process

1. Empathy: Understanding the user's needs.
2. Define: Identifying the problem.
3. Ideate: Exploring various possible solutions.
4. Prototype: Creating low-scale representations of solutions.
5. Test: Testing each solution and refine.
6. Implement: Implementation of the solution in real-life.

Objective

The ProFind mobile application is designed to streamline the hiring process for SMME businesses in South Africa while simplifying job applications for candidates.
For business owners, ProFind offers tools to receive real-time notifications when candidates match job requirements, efficiently shortlist potential hires, and forward CVs with ease. On the applicant side, users can apply to multiple job postings without the need to create a user profile. The app allows for quick CV uploads, real-time confirmation when a CV has been received, and live updates on the status of each application.


User Personas

We created user personas for both businesses and applicants, based on their goals, challenges, and behavioral traits. These personas played a key role in shaping the design, helping us empathise with users and make informed decisions. Journey Map

We developed a journey map to visualize the end-to-end experience for both businesses and applicants throughout the job application process. This helped us identify key touchpoints, pain points, and opportunities for improvement.


Wireframes

High-fidelity wireframes were created and used for user testing, allowing us to validate the layout, functionality, and overall user experience before moving into development. This helped identify usability issues early and ensured the design aligned with user expectations.
